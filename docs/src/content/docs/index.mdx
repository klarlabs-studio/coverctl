## The agent loop

```text
You ─► Agent edits ─► coverctl check (MCP) ─► passed:false / failing:api
                              ▲                          │
                              │                          ▼
                              └─ Agent fixes ◄── coverctl suggest api (MCP)
                                                         │
                                                         ▼
                                                  Commit lands clean
                                              CI never sees the regression
```

Below: an actual agent session shape after `coverctl mcp serve` is wired into Claude Code. *Rendered for clarity — your client may format tool-calls differently.*

```text
You:    Add the email-validation feature to the api domain.

Claude: I'll add EmailValidator + tests in internal/api/email.

  [edits: internal/api/email.go, internal/api/email_test.go]

  Tool call → coverctl check
  Result   → passed: false; failing: api (78.2%, required 80%)
             uncovered: internal/api/email.go:42-58 (error paths)

  Tool call → coverctl suggest api
  Result   → uncovered files: email.go (16 lines), retry.go (4 lines)

  Two error branches in EmailValidator.Validate are uncovered.
  I'll add cases for empty-input and malformed-domain so api meets
  the 80% threshold before we commit.
```

The regression that used to surface in CI 8 minutes later is fixed in the same agent turn.

<Aside type="caution" title="Watch for these agent failure modes">

- **Threshold lowering as a "fix"** — agents proposing edits to `.coverctl.yaml` to drop a domain's `min:`. That's not a fix.
- **Hallucinated coverage numbers** — claiming coverage rose without re-running `coverctl check`.
- **Ignoring rejection codes** — retrying the same input after a structured rejection. The `remediation` field tells the agent what to change.

coverctl returns deterministic structured signals; the agent's *fix* still needs a human reading.

</Aside>

## Why not your current setup?

| Today | Why it fails for agent-edited code |
|---|---|
| **Red-CI agent loop** (status quo) | Agent edits → tests fail in CI 8 minutes later → context lost. coverctl: feedback in seconds, in the same turn. |
| **Codecov / Coveralls dashboards** | Post-merge visibility for humans. coverctl: pre-commit signal for agents. |
| **Native `go test -cover` / `pytest --cov`** | Per-language only, no domain policy, no agent-callable interface. coverctl: uniform across 15 languages, MCP-native. |

## Quick Start

<LinkCard
  title="Wire coverctl into Claude Code, Cursor, or Cline →"
  description="Connect through MCP. Agent calls check, suggest, debt directly inside the edit loop, before commit. Most users start here."
  href="/coverctl/quick-start-agent/"

Working from a terminal instead? [Terminal quickstart →](/coverctl/quick-start/)

```bash
# Install
brew install felixgeelhaar/tap/coverctl   # or: go install github.com/felixgeelhaar/coverctl@latest

# One-time project setup
coverctl init

# Then either run from terminal:
coverctl check

# ...or wire into Claude Code / Cursor / Cline (default agent mode)
# and let the agent invoke coverctl during editing.
```
